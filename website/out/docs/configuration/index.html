Configuration

Complete guide to configuring Quantum Code for your environment, including API keys, analysis settings, and output preferences.

Configuration Options

API Keys

OPENAI_API_KEY

OpenAI GPT-4 API key for advanced analysis

Example:
sk-...
ANTHROPIC_API_KEY

Anthropic Claude 3 API key for reasoning tasks

Example:
sk-ant-...
GOOGLE_API_KEY

Google Gemini Pro API key for fast analysis

Example:
AIza...

Analysis Settings

QUANTUM_CODE_MODELS

Comma-separated list of AI models to use

Example:
gpt-4,claude-3,gemini-pro
QUANTUM_CODE_THRESHOLD

Confidence threshold for recommendations (0.0-1.0)

Example:
0.8
QUANTUM_CODE_TIMEOUT

API request timeout in seconds

Example:
30

Output Settings

QUANTUM_CODE_OUTPUT_FORMAT

Default output format (text, json, yaml)

Example:
json
QUANTUM_CODE_VERBOSE

Enable verbose output (true/false)

Example:
false
QUANTUM_CODE_OUTPUT_DIR

Directory to save analysis reports

Example:
./reports

YAML Configuration File

Create a comprehensive configuration file for advanced setups

config.yaml

# Quantum Code Configuration
# Save this file as ~/.quantum-code/config.yaml

# API Keys (get from respective providers)
openai_api_key: "sk-your-openai-key"
anthropic_api_key: "sk-ant-your-anthropic-key"
google_api_key: "your-google-api-key"

# Analysis Configuration
models:
  - gpt-4
  - claude-3
  - gemini-pro

# Analysis Settings
threshold: 0.8
timeout: 30
max_tokens: 4000

# Output Configuration
output_format: json
verbose: false
output_dir: ./reports

# Advanced Settings
consensus_algorithm: weighted_average
parallel_requests: true
cache_results: true

Save this file as ~/.quantum-code/config.yaml or specify with --config flag

Environment Variables Setup

1

Create Environment File

Create a .env file in your project root

# .env
OPENAI_API_KEY=sk-your-openai-key
ANTHROPIC_API_KEY=sk-ant-your-anthropic-key
GOOGLE_API_KEY=your-google-api-key

QUANTUM_CODE_MODELS=gpt-4,claude-3,gemini-pro
QUANTUM_CODE_THRESHOLD=0.8
QUANTUM_CODE_OUTPUT_FORMAT=json
2

Load Environment Variables

Load the .env file in your shell or application

# For bash/zsh
source .env

# For fish shell
source .env.fish

# For Windows PowerShell
. .\.env
3

Verify Configuration

Test that your configuration is working

quantum-code --version
quantum-code configure --check

Security Best Practices

Important security considerations for your API keys and configuration

✅ Do's

  • Use environment variables for API keys
  • Add .env files to .gitignore
  • Use secure key management services
  • Rotate API keys regularly

❌ Don'ts

  • Hardcode API keys in source code
  • Commit .env files to version control
  • Share API keys in logs or outputs
  • Use weak or easily guessable keys
